# Phased Delivery Plan

## Phase 1 - Guided Core Experience
### Goals
- Solve Event vs Instance confusion
- Prevent common setup mistakes
- Provide event-first launcher and guided workflows
- Improve attendee visibility and export

### Deliverables
1. FIU Events app shell
2. Landing page / launcher with:
   - Create New Event
   - Clone Existing Event
   - Add New Instance to Existing Event
3. Guided Create New Event workflow
4. Robust Clone Event workflow with warnings and review steps
5. Guided Add New Instance workflow
6. Event/Instance operational readiness indicators
7. Instance attendee/attendance manager
8. CSV export for attendees
9. Admin/read-only diagnostic visibility into payment/reminder/Opportunity creation setup

## Phase 2 - Specialized and Advanced Operations
### Goals
- Support OGI/study abroad workflows
- Incorporate more advanced Summit feature usage
- Expand operational tooling and exception handling

### Possible Deliverables
- OGI-specific guided experience
- Better support for Summit Questions and Summit Emails users
- Advanced payment-specific UX
- Advanced reminder routing and larger-scale messaging patterns
- Reporting/exception dashboarding

## Implementation Order Recommendation
1. Discovery from Prod
2. App shell and launcher in Partial UAT
3. Create/Clone/Add-Instance workflows
4. Attendee manager and CSV export
5. Operational diagnostics
6. Reminder and email enhancement opportunities
